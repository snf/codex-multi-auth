# Code of Conduct

## Our Standard

This project expects respectful, technically grounded collaboration.

Examples of behavior that help the project:

- being respectful and constructive in issues, pull requests, and discussions
- reporting bugs with clear reproduction steps and real evidence
- keeping feedback focused on behavior, code, and documentation
- respecting the project's policy and security boundaries

Examples of unacceptable behavior:

- harassment, abuse, or hostile communication
- requests for credential theft, token scraping, auth bypasses, or policy evasion
- sharing secrets, tokens, or other sensitive user data
- deliberately misleading bug reports or review comments

## Scope

This code of conduct applies to the repository, issue tracker, pull requests, and project discussions.

## Enforcement

Project maintainers may remove, edit, or reject comments, issues, pull requests, or other contributions that violate this code of conduct.

## Reporting

For conduct concerns, contact the maintainer through the repository's GitHub contact channel.

For security issues, do not open a public issue. Follow [SECURITY.md](SECURITY.md).
