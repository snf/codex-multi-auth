# Release v0.1.0-beta.0

Release date: 2026-02-27  
Type: prerelease (`beta`)

## Highlights

- Introduces a Codex-first multi-account auth experience through `codex auth ...`.
- Adds a full interactive TUI dashboard for account login, switch, health checks, and forecasting.
- Adds advanced reliability controls for refresh/retry/failover/sync behavior.
- Adds extensive test coverage and documentation overhaul for beginner-first onboarding.

## Added

- New account management command surface:
  - `codex auth login`
  - `codex auth list`
  - `codex auth switch <index>`
  - `codex auth check`
  - `codex auth forecast`
  - `codex auth fix`
  - `codex auth doctor`
  - `codex auth report`
  - `codex auth features`
- New interactive TUI modules:
  - account list and detail actions
  - quick health checks
  - forecast and safe auto-fix flows
  - settings hub (display/theme/behavior/backend categories)
- New backend/runtime modules:
  - session affinity
  - live account sync
  - preemptive quota scheduler
  - refresh guardian and lease protection
  - capability policy and entitlement cache
  - stream failover and failure policy
  - runtime paths and unified settings model

## Changed

- Storage and migration behavior hardened:
  - improved legacy path migration handling
  - backup and WAL handling improvements
  - stronger recovery behavior for account files
- OAuth and auth menu behavior refined:
  - clearer login mode handling
  - improved browser/manual flow behavior
  - better source-index resolution feedback
- Prompt/cache handling improved:
  - safer prompt source metadata handling
  - retry/backoff for transient filesystem contention

## Fixed

- Reduced OAuth URL leakage risk in logs by redacting sensitive query values before logging.
- Removed TOCTOU command-check pattern in browser opener path.
- Added retry/backoff handling for transient Windows `EBUSY`/`EPERM` contention in critical paths.
- Resolved numerous UX and consistency issues reported in PR review and TUI iteration.

## Documentation

- Reworked docs structure for beginner-first navigation.
- Added full command, settings, and storage path references.
- Expanded troubleshooting, privacy, and upgrade guidance.

## Quality and Validation

- Lint/typecheck/test gates pass on merged state:
  - `npm run lint`
  - `npm run typecheck`
  - `npm test`
- Test status at merge:
  - `79` test files
