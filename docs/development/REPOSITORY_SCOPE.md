# Repository Scope Map

This document is the canonical map of repository scope, file ownership, and extension points.

## Top-Level Scope

| Path | Purpose | Source of Truth |
| --- | --- | --- |
| `index.ts` | Plugin entrypoint, provider hook wiring, fetch orchestration, tool registration | Yes |
| `lib/` | Core runtime logic (auth, rotation, request/response transforms, storage, UI, prompts) | Yes |
| `test/` | Vitest unit/integration/property coverage | Yes |
| `scripts/` | Build/install/helper scripts | Yes |
| `config/` | Copy-paste OpenCode config templates | Yes |
| `docs/` | User + developer documentation | Yes |
| `assets/` | Static project assets | Yes |
| `dist/` | Build output generated by `npm run build` | No (generated) |

## Core Module Ownership

| Area | Main Path(s) | Responsibility |
| --- | --- | --- |
| Plugin orchestration | `index.ts` | Request pipeline, routing, retries, tool surface |
| OAuth + token lifecycle | `lib/auth/auth.ts`, `lib/auth/server.ts`, `lib/refresh-queue.ts` | Login flow, callback handling, refresh control |
| Multi-account rotation | `lib/accounts.ts`, `lib/rotation.ts`, `lib/health.ts`, `lib/circuit-breaker.ts` | Account selection, health scoring, failover |
| Rate limits and retries | `lib/accounts/rate-limits.ts`, `lib/request/rate-limit-backoff.ts` | Quota awareness, wait/retry logic |
| Request transformation | `lib/request/request-transformer.ts`, `lib/request/helpers/model-map.ts` | OpenCode request normalization to Codex backend format |
| Fetch + error mapping | `lib/request/fetch-helpers.ts` | Headers, timeout handling, backend error normalization |
| Stream/SSE handling | `lib/request/response-handler.ts` | Streaming parsing and terminal response assembly |
| Prompt bridge | `lib/prompts/codex.ts`, `lib/prompts/codex-opencode-bridge.ts`, `lib/prompts/opencode-codex.ts` | Prompt templates, codex bridge instructions, sync/cache |
| Config loading | `lib/config.ts`, `lib/schemas.ts`, `config/*.json` | Plugin options, schema validation, templates |
| Storage and migrations | `lib/storage.ts`, `lib/storage/paths.ts`, `lib/storage/migrations.ts` | Account persistence, path resolution, schema upgrades |
| Recovery/session state | `lib/recovery/` | Conversation recovery state and persistence |
| Codex CLI sync | `lib/codex-cli/` | Codex CLI account/state synchronization |
| Tooling | `lib/tools/hashline-tools.ts`, `lib/table-formatter.ts` | User-facing plugin tools and CLI display |

## AGENTS Scope Hierarchy

AGENTS files apply hierarchically. Deeper AGENTS files override parent rules for their subtree.

| AGENTS file | Scope |
| --- | --- |
| `AGENTS.md` | Entire repository |
| `lib/AGENTS.md` | `lib/**` |
| `test/AGENTS.md` | `test/**` |

## Generated/Local Artifacts (Out of Scope for Source Edits)

| Path Pattern | Notes |
| --- | --- |
| `dist/**` | Generated build output, do not edit by hand |
| `.tmp*`, `tmp*` | Local scratch/release artifacts |
| `.omx/**`, `.sisyphus/**` | Local agent/runtime state |
| `coverage/**` | Generated test coverage output |
| `node_modules/**` | Installed dependencies |

## Extension Points for New Work

Use this checklist when adding a new feature (for example a Codex CLI-focused config/plugin extension).

1. Update plugin behavior and registration in `index.ts`.
2. Add/adjust option parsing in `lib/config.ts` and validation in `lib/schemas.ts`.
3. Update request transform and model mapping in `lib/request/request-transformer.ts` and `lib/request/helpers/model-map.ts`.
4. Add prompt behavior in `lib/prompts/` when bridge instructions change.
5. Add tests in `test/` for success path, edge cases, and regressions.
6. Update user docs (`README.md`, `docs/configuration.md`, `docs/getting-started.md`) and developer docs (`docs/development/*.md`).
